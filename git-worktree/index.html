优点：

1.因为所有工作目录共享一个仓库，所以一个更新意味着整个更新（A 目录里对分支做的改动，B 目录里切到此分支也是改动后的；避免到时候找不到某个未推送的改动改到了哪个仓库）
2.可以快速进行并行开发，同一个项目多个分支同时并行演进
3.可避免node_modules冲突
4.相比单纯clone/copy，可使用命令更方便直观的管理

注意：使用 git worktree 创建的多个目录，不能有任何两个目录在同一个分支下
